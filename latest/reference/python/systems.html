Available system implementation¶

rascaline.systems.wrap_system(system)¶

This function is automatically called on all systems passed to rascaline.calculators.CalculatorBase.compute(). It wraps different systems implementation into the right class to make them compatible with rascaline. The following system types are supported:

  • ase.Atoms: the Atomistic Simulation Environment Atoms class

  • chemfiles.Frame: chemfiles’ Frame type

If system is already a subclass of rascaline.SystemBase, it is returned as-is.

Parameters

system – external system of one of the above type

Returns

a specialized instance of rascaline.SystemBase

class rascaline.systems.AseSystem(atoms)¶

This class implements rascaline.SystemBase using the ase.Atoms to get the data and ase.neighborlist.neighbor_list to compute the neighbor list.

Parameters

atoms – ase.Atoms object to be wrapped in this AseSystem

class rascaline.systems.ChemfilesSystem(frame)¶

This class implements rascaline.SystemBase wrapping a chemfiles.Frame. Since chemfiles does not offer a neighbors list, this implementation of system can only be used with use_native_system=True in rascaline.calculators.CalculatorBase.compute().

Atomic species are assigned as the atomic number if the atom type is one of the periodic table elements; or as a value above 120 if the atom type is not in the periodic table.

Parameters

frame – chemfiles.Frame object to be wrapped in this ChemfilesSystem

class rascaline.SystemBase¶

Base class implementing the System trait in rascaline. Developers should implement this class to add new kinds of system that work with rascaline.

Most users should use one of the already provided implementation, such as rascaline.systems.AseSystem or rascaline.systems.ChemfilesSystem instead of using this class directly.

cell()¶

Get the 3x3 matrix representing unit cell of the system. The cell should be written in row major order, i.e. [[ax, ay, az], [bx, by, bz], [cx, cy, cz]], where a/b/c are the unit cell vectors.

compute_neighbors(cutoff)¶

Compute the neighbor list with the given cutoff, and store it for later access using rascaline.SystemBase.pairs() or rascaline.SystemBase.pairs_containing().

pairs()¶

Get all neighbor pairs in this system as a list of tuples (int, int, float, (float, float, float)) containing the indexes of the first and second atom in the pair, the distance between the atoms, and the wrapped between them. Alternatively, this function can return a numpy array with dtype=rascal_pair_t.

The list of pair should only contain each pair once (and not twice as i-j and j-i), should not contain self pairs (i-i); and should only contains pairs where the distance between atoms is actually bellow the cutoff passed in the last call to rascaline.SystemBase.compute_neighbors().

This function is only valid to call after a call to rascaline.SystemBase.compute_neighbors() to set the cutoff.

pairs_containing(center)¶

Get all neighbor pairs in this system containing the atom with index center.

The same restrictions on the list of pairs as rascaline.SystemBase.pairs() applies, with the additional condition that the pair i-j should be included both in the list returned by pairs_containing(i) and pairs_containing(j).

positions()¶

Get the cartesian position of all atoms in this system. The returned positions must be convertible to a numpy array of shape (self.size(), 3).

size()¶

Get the number of atoms in this system as an integer

species()¶

Get a list of integers or a 1D numpy array of integers containing the atomic species for each atom in the system. Each different atomic species should be identified with a different value. These values are usually the atomic number, but don’t have to be.
